# Final Proposal
### Uno clone

For this final I am planning to try to replicate Uno inside a website. This idea came to me as I looked at the requirments of the final with the idea of making a game, mainly when I saw that we needed to include drag and drop. I also found the idea feasable as a game I could actually make using html, javascript, and c#. I feel that uno also works very well for this project due to its simple gameplay loop that has some intracacies in how and when you can play cards. The familiarity with the idea also means that the requirmenets I need for getting the game to work are easily understandable.

My plan is to start by getting a home page that allows you to start a game that then takes you to a different page where you actually play uno. From there I plan to work on my c# api to allow the player to play cards of the according color or draw cards if they can't play a card. After I have all the base mechanics working I plan to add multiplayer capability by having a player generate a room id when they start a game on the home page. After I add base mechanics and multiplayer I then plan to move onto implementing specialty cards such as draw cards, reverse cards, and skip cards.
